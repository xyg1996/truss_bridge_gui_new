Outil en ligne de commande¶

L’application AsterStudy contient l’exécutable asterstudy_assistant*. Il permet de générer et tester un *assistant de calcul*, et également de produire des fichiers **Code_Aster COMM à partir des templates. Les fichiers COMM peuvent être importés dans l’application AsterStudy.

Les paramètres pour utiliser asterstudy_assistant en ligne de commande sont les suivants :

usage: asterstudy_assistant [-h] [-v] command [args]

AsterStudy assistant helper.

optional arguments:
  -h, --help     show this help message and exit
  -v, --version  show program's version number and exit

Supported commands:
  generate    Generate assistant.
  pack        Pack assistant to archive.
  run         Run assistant.
  verify      Verify assistant.

Les options de chaque commande sont décrites dans les paragraphes suivants.

Générer l’assistant¶

La commande generate peut être utilisée pour générer le fichier wizard.py à partir du fichier de déclaration declaration.json. Par défaut, le fichier wizard.py est créé dans le répertoire où se trouve le fichier declaration.json. Il est possible de spécifier le chemin du fichier déclaration, et également celui où créé le fichier de l’assistant.

La syntaxe de la commande generate est la suivante :

usage: asterstudy_assistant generate [-h] [-d FILE] [-w FILE] folder

Generate calculation assistant wizard.

positional arguments:
  folder                       directory with the wizard description file

optional arguments:
  -h, --help                   show this help message and exit
  -d FILE, --declaration FILE  declaration file (default: declaration.json)
  -w FILE, --wizard FILE       wizard file (default: wizard.py)

Exemple :

asterstudy_assistant generate /home/user/asterstudy/my_assistant

Lancer l’assistant¶

La commande run affiche un assistant à partir d’un fichier généré wizar.py, ou directement à partir du fichier déclaration declaration.json.

Lorsque l’utilisateur termine l’assistant (en pressant le bouton Fin), celui-ci analyse les paramètres de fichier template (par défaut template.comm), substitue ces paramètres avec les valeurs saisies dans l’assistant, et génère un fichier de sortie (par défaut stage.comm), qui peut être importé par l’application AsterStudy.

La syntaxe de la commande run est la suivante :

usage: asterstudy_assistant run [-h] [-d FILE] [-w FILE] [-t FILE] [-o FILE] folder

Run calculation assistant wizard.

positional arguments:
  folder                       wizard directory

optional arguments:
  -h, --help                   show this help message and exit
  -d FILE, --declaration FILE  declaration file; needed only if wizard file does not
                               exist, in which case wizard is generated from description
                               (default: declaration.json)
  -w FILE, --wizard FILE       wizard file (default: wizard.py)
  -t FILE, --template FILE     template file (default: template.comm)
  -o FILE, --output FILE       output comm file; relative path is interpreted as a
                               sub-path within wizard directory (default: stage.comm)

Exemple :

asterstudy_assistant run /home/user/asterstudy/my_assistant

Vérifier un assistant¶

La commande verify peut être utilisée pour vérifier la validité d’un assistant de calcul.

Avertissement

Cette commande n’est pas encore implémentée !

Packager un assistant¶

La commande pack crée une archive ZIP contenant l’assistant de calcul.

La syntaxe de la commande pack est la suivante :

usage: asterstudy_assistant pack [-h] [-d FILE] [-w FILE] [-t FILE] [-o FILE] folder

Pack calculation assistant wizard to archive.

positional arguments:
  folder                       wizard directory

optional arguments:
  -h, --help                   show this help message and exit
  -d FILE, --declaration FILE  declaration file (default: declaration.json)
  -w FILE, --wizard FILE       wizard file (default: wizard.py)
  -t FILE, --template FILE     template file (default: template.comm)
  -o FILE, --output FILE       output archive file; relative path is interpreted
                               as a sub-path within wizard directory; if not
                               specified, default file name is used (default: None)

Exemple :

asterstudy_assistant pack /home/user/asterstudy/my_assistant

Fichier déclaration¶

Après avoir écrit le fichier template, vous pouvez configurer l’IHM de l”assistant de calcul en écrivant le fichier déclaration.

Le fichier déclaration est un fichier texte au format JSON. Il fournit une description des pages qui constituent l’assistant, les paramètres à afficher sur chaque page, les contraintes sur certains contrôles graphiques, ainsi que les attributs génériques de l’assistant.

Structure du fichier déclaration¶

En général, la structure du fichier déclaration est la suivante :

{
  "title": "Title of calculation assistant",
  "pages": [
    {
      "title": "Title of page 1",
      "parameters": [
        {
          "name": "PARAMETER_1_IN_TEMPLATE_FILE",
          "attribute_1": "Value of attribute_1",
          "attribute_2": "Value of attribute_2",
          ...
        }
      ]
    },
    {
      "title": "Title of page 2",
      "parameters": [
        {
          "title": "Title of group 1 on page 2",
          "parameters": [
            {
              "name": "PARAMETER_2_IN_TEMPLATE_FILE",
              "attribute_1": "Value of attribute_1",
              ...
            },
            {
              "name": "PARAMETER_3_IN_TEMPLATE_FILE",
              "attribute_1": "Value of attribute_1",
              ...
            }
          ]
        },
        {
          "title": "Title of group 2 on page 2",
          "parameters": [
            {
              "name": "PARAMETER_4_IN_TEMPLATE_FILE",
              "attribute_1": "Value of attribute_1",
              ...
            },
            {
              "name": "PARAMETER_5_IN_TEMPLATE_FILE",
              "attribute_1": "Value of attribute_1",
              ...
            }
          ]
        },
        ...
      ]
    },
    ...
  ]
}

La structure du fichier déclaration est décrite dans la table ci-dessous. Chaque élément est décrit en tant que dictionnaire avec une liste de paramètres nommés. Les séquences représentant un ensemble d’entités dans un élément sont représentées sous forme de listes. L’élément racine représente l’assistant lui-même.

Le groupe de contrôles peuvent être imbriqués ; le premier niveau représente les pages de l’assistant.

Wizard
pages Page 1
parameters Param 1
  Param 2
  …
Page 2
parameters Group 1
  parameters Param K
    Param K+1
    …
  Group 2
  parameters …
  Param N
  Param N+1
  …
…    

Les attributs supportés par chaque élément sont décrits dans les tables ci-dessous.

Wizard attributes¶
Attribute Type Description Obligatoire
format chaîne Version du format (actuellement le format supporté est 2.1) oui
title chaîne Titre principal de l’assistant, visible sur chaque page. oui
class chaîne

Le nom de la classe de l’assistant généré.

Par défaut : Wizard.

 non
description chaîne

Description de l’assistant de calcul.

Par défaut : none.

 non
pages liste Liste des pages décrivant les étapes de la saisie. oui

Attributs de page¶
Attribute Type Description Obligatoire
title chaîne

Titre de la page, affiché sous le titre principal.

Par défaut : none.

 non
description chaîne

Description de la page affichée en infobulle.

Par défaut : none.

 non
parameters liste Liste des paramètres pour la saisie utilisateur. Il est possible de créer des sous listes pour grouper les widgets. oui

Attributs des groupes¶
Attribute Type Description Obligatoire
title chaîne

Titre du groupe. Si non précisé, un groupe sans titre sera créé.

Par défaut : none.

 non
description chaîne

Description supplémentaire pour le groupe, affichée dans une infobulle.

Par défaut : none.

 non
parameters liste Liste de paramètres pour la saisie utilisateur. Il est possible d’avoir des listes imbriquées pour créer des groupes de widgets. oui
name chaîne S’applique uniquement au type de paramètre choices (voir ci-dessous). Spécifie un identifiant unique lié à un choix particulier. oui

Attributs des paramètres¶
Attribute Type Description Obligatoire
name chaîne Nom du paramètre. Identifiant unique utilisé pour faire la correspondance avec le template de l’étape. oui
title chaîne

Titre du paramètre.

Par défaut : none

 non
description chaîne

description supplémentaire pour le paramètre, affichée en tant qu’infobulle.

Par défaut : none.

 non
typ chaîne

Type de widget :

  • « int » : si l’attribut into est spécifié, une liste déroulante est affichée (voir ci-dessous), sinon un champ simple ;
  • « float » : si l’attribut into est spécifié, une liste déroulante est affichée (voir ci-dessous), sinon un champ simple ;
  • « bool » : afficher le widget de case à cocher;
  • « string » : si l’attribut into est spécifié, une liste déroulante est affichée (voir ci-dessous), sinon un champ simple ;
  • « table » : affiche le widget table, dont le contenu est spécifié avec l’attribut columns (voir ci-dessous) ;
  • « file » : affiche le widget de sélection de fichier ;
  • « mesh » : affiche le widget de sélection de maillage ;
  • « group_ma » : affiche le widget de sélection de groupe de maille pour sélectionner un groupe d’éléments ;
  • « group_no » : affiche le widget de sélection de groupe de maille pour sélectionner un groupe de noeuds ;
  • « choice » : affiche une liste de radio boutons, un seul choix peut être sélectionné. Chacun des choix peut afficher des widgets supplémentaires.
  • « text » : affiche une zone de texte en lecture seule, utile pour une longue description.
  • « hidden » : n’affiche rien, utile pour les choix avec aucun paramètre.

Par défaut : « string ».

 non
default any

Valeur par défaut du widget. Le type de valeur dépend du type d’entrée (int, float, bool, etc.).

Par défaut : none.

 non
into liste

Disponible si la valeur de typ est int, float ou string. Lorsqu’il est défini, génère un widget liste déroulante qui contient les valeurs spécifiées comme choix possibles. Le type de valeur dans la liste dépend du type d’entrée (int, float, string).

Par défaut : none.

 non
val_min int, float

Disponible si la valeur de typ est int ou float et que into n’est pas spécifié. Spécifie la valeur minimale autorisée pour le paramètre.

Par défaut : none.

 non
val_max int, float

Disponible si la valeur de typ est int ou float et que into n’est pas spécifié. Spécifie la valeur maximale autorisée pour le paramètre.

Par défaut : none.

 non
regex chaîne

Disponible si la valeur de typ est string et que into n’est pas spécifié. Spécifie le format de la valeur du paramètre en tant qu’expression régulière, par exemple « [a-zA-Z]\w{1,7} ».

Note

Les caractères “” doivent être doublé pour permettre un traitement correct du fichier JSON.

Par défaut : none.

 non
length chaîne

Disponible si la valeur de typ est string et que into n’est pas spécifié. Spécifie la longueur de la valeur.

Note

length a une priorité moins importante que l’attribut regex.

Par défaut : none.

 non
len_min int

Disponible si la valeur de typ est string et que into n’est pas spécifié. Spécifie la longueur minimale de la valeur.

Note

Les attributs len_min et len_max (voir ci-dessous) ont une priorité inférieure aux attributs regex et length.

Par défaut : none.

 non
len_max int

Disponible si la valeur de typ est string et que into n’est pas spécifié. Spécifie la longueur maximale de la valeur.

Note

Les attributs len_min (voir ci-dessus) et len_max ont une priorité inférieure aux attributs regex et length.

Par défaut : none.

 non
text chaîne Disponible si typ est text. Le texte est donné sous forme d’une string ou d’une liste de string pour une longue description sur plusieurs lignes. non
columns liste

Disponible si la valeur de typ est table. Spécifie les colonnes de la table. Chaque colonne est spécifiée en tant que parameter, les mêmes attributs sont supportés.

Les types de valeurs suivants sont pris en charge en tant qu’entrée dans les colonnes de table:

  • « int »
  • « float »
  • « chaîne »
  • « group_ma »
  • « group_no »

Deux attributs supplémentaires sont disponibles :

  • « group_ma » : permet de sélectionner plusieurs groupe d’éléments ;
  • « groups_no » : permet de sélectionner plusieurs groupes de noeuds.
 oui
mode chaîne

Disponible si la valeur de typ est file. Spécifie le mode utilisé pour le fichier. Les valeurs possibles sont :

  • in : pour les fichiers d’entrée ; une fenêtre de sélection de fichier est affichée, le fichier doit exister ;
  • out : pour les fichiers de sortie ; une fenêtre de sauvegarde de fichier est affichée, le fichier peut ne pas exister.

Par défaut : « in »

 non
filters chaîne

Disponible si la valeur de typ est fichier. Spécifie le filtre de fichier pour les fenêtres de sélection ou de sauvegarde de fichier. Pour spécifier plusieurs filtres, il faut les séparer en utilisant deux points virgules (« ;; »).

Par défaut : none.

 non
unit int

Disponible si la valeur de typ est file ou mesh. Il s’agit de l’unité par défaut associée avec le fichier.

Par défaut : none.

 non
mesh chaîne

Disponible si la valeur de typ est group_ma, group_no ou table (dans ce cas, seulement pour les colonnes dont le typ est group_ma, group_no, groups_ma ou groups_no). Spécifie le nom du paramètre de maillage où trouver la liste des groupes. Si non spécifié, le premier paramètre du maillage est utilisé.

This attribute is mandatory but it can not be checked in table parameters.

Par défaut : none.

 oui
rule chaîne

Disponible si la valeur de typ est table. Spécifie une règle de validation personnalisée pour chaque ligne de la table.

Par défaut : none.

 non
choices liste Disponible si la valeur de typ est choice. Spécifie la liste des choix possibles. Chaque choix est représenté par un bouton radio. Lorsqu’il est sélectionné, le bouton radio peut activer des paramètres supplémentaires. Ainsi, le type choices est un cas particulier de group. oui
mandatory bool

S’applique uniquement aux colonnes de table. Lorsque défini sur true, nécessite que la colonne correspondante soit explicitement renseignée par l’utilisateur.

Par défaut : true.

 non

Lien avec le catalogue¶

Avertissement

Il s’agit d’une extension de l’implémentation de base de l”assistant de calcul qui n’a pas encore été implémentée. Ce paragraphe doit être considéré comme un aperçu de cette future fonctionnalité.

Avec l’approche générale, la création d’un fichier déclaration nécessite beaucoup d’efforts, et celui-ci peut devenir assez volumineux, car tous les attributs nécessaires doivent être spécifiés pour chaque paramètre de l’assistant de calcul.

Pour simplifier la création du fichier déclaration, il est possible de lier n’importe quel paramètre directement au catalogue Code_Aster.

Cela peut être réalisé grâce à l’attribut supplémentaire url :

Attribute Type Description Obligatoire
url chaîne

Lien avec le catalogue sous la forme d’un chemin vers un mot-clé particulier du catalogue des commandes.

Exemple : AFFE_MODELE/AFFE/PHENOMENE

 oui

Cet attribut permet d’extraire automatiquement tous les attributs nécessaires pour configurer le contrôle graphique (valeur minimum, maximum, par défaut, liste des valeurs possibles, type de valeur, etc).

Note

Il est toujours possible de surcharger la description du paramètre extraite depuis le catalogue en utilisant les attributs décrits au paragraphe précédent.

Exemple :

...
"parameters": [
  {
    "name": "UNITE",
    "url": "LIRE_MAILLAGE/UNITE"
  },
  {
    "name": "PHENOMENE",
    "url": "AFFE_MODELE/AFFE/PHENOMENE",
    "into": ["MECANIQUE", "THERMIQUE"]
  }
]
...

Dans cet exemple, les attributs du paramètre UNITE de l’assistant sont automatiquement extraits de la commande LIRE_MAILLAGE du catalogue Code_Aster.

De même, les attributs du paramètre PHENOMENE seront extraits du mot-clé correspondant de la commande du catalogue AFFE_MODELE ; toutefois, l’attribut « into » étant redéfini dans le fichier déclaration, la plage de valeurs possibles sera limitée à MECANIQUE et THERMIQUE uniquement (les autres valeurs possibles indiquées dans le catalogue seront ignorées).

Voir un exemple plus détaillé dans la Section exemple.

Extraire la description depuis le template¶

Avertissement

Il s’agit d’une extension de l’implémentation de base de l”assistant de calcul qui n’a pas encore été implémentée. Ce paragraphe doit être considéré comme un aperçu de cette future fonctionnalité.

Une autre extension de la fonctionnalité de base offre la possibilité de spécifier les paramètres de l’assistant de calcul sans décrire aucun attribut dans le fichier déclaration. Dans ce cas, l’assistant de calcul détectera automatiquement la commande Code_Aster et un mot clé pour chaque paramètre en analysant le fichier modèle.

Le fichier déclaration reste nécessaire pour spécifier la répartition des paramètres sur les différentes pages de l’assistant, ou au sein de groupes.

Voir la Section exemples.

Description générale¶

L’interface utilisateur de l”assistant de calcul se présente sous la forme d’un assistant, composé d’une suite de pages qui contiennent des champs de saisie. L’utilisateur renseigne les paramètres du cas d’étude en naviguant entre ces pages et en saisissant les valeurs dans les champs correspondants.

L’interface de chaque assistant est automatiquement générée par AsterStudy depuis la description spécifiée dans le fichier de déclaration. En plus de cette approche dynamique, ile est possible de générer une implémentation statique de l’interface de l”assistant afin de pouvoir la personnaliser (comme par exemple implémenter des règles de vérification, utiliser des widgets spécifiques, ajouter une logique métier entre différents contrôles, etc).

Lorsque l’utilisateur presse le bouton Fin sur la dernière page de l’assistant, celui-ci va créer une nouvelle étape dans l’étude, en insérant les valeurs saisies par l’utilisateur dans le fichier template fourni avec l”assistant et en important le résultat dans AsterStudy.

Assistant de calcul

En général, chaque assistant de calcul est représenté par un répertoire comprenant une liste de fichiers :

  • fichier template, décrivant une étape paramétrée dans le langage Code_Aster ;
  • fichier déclaration, décrivant l’interface de l’assistant de calcul ;
  • option, implémentation de l’assistant, qui contient une personnalisation supplémentaire de l’interface ;
  • option, fichier de configuration ;
  • option, fichier de ressources.

Dans cette liste, seuls les fichiers template et déclaration sont obligatoires. Outre les fichiers répertoriés, d’autres fichiers peuvent également être présents. Une fois généré à partir du fichier déclaration, l’implémentation de l’assistant peut être librement étendue pour répondre aux besoins particuliers de l’assistant.

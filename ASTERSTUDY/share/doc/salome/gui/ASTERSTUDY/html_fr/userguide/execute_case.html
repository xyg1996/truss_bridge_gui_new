Exécuter un cas¶

Pour exécuter un cas, il faut se placer dans la vue Historique.

Sélection des étapes à exécuter¶

Avant de lancer le calcul, l’utilisateur doit choisir les étapes à exécuter. Pour cela, il doit cliquer sur les icônes des étapes qu’il souhaite exécuter dans son calcul :

  • pour exécuter l’étape,
  • pour ignorer l’étape,
  • pour réutiliser les résultats du calcul précédent pour cette étape (voir ci-après).

Note

Si une étape est sélectionnée pour exécution, les étapes précédentes sont également automatiquement sélectionnées.

De même, si une étape est marquée ignorée, les étapes qui suivent sont ignorées.

Par défaut, s’il existe des résultats précédents ceux-ci sont réutilisés.

Le bouton permet à l’utilisateur de spécifier s’il souhaite conserver les résultats d’un précédent calcul. Lorsque ce bouton est pressé, les résultats de l’étape correspondant sont stockés séparément, sinon ils sont ajoutés aux résultats de l’étape suivante.

Le panneau Paramètres d’exécution permet à l’utilisateur de définir les paramètres d’exécution, comme le serveur à utiliser, la taille maximale de mémoire, un temps d’exécution maximal…

Fenêtre des paramètres d’exécution

Pour démarrer un calcul, l’utilisateur doit

  • Sélectionner le Cas courant dans le panneau Cas,
  • Sélectionner les options d’exécution pour chaque étape (voir ci-dessus),
  • Préciser les paramètres d’exécution dans le panneau Paramètres d’exécution,
  • Appuyer sur le bouton Run.

Les étapes sélectionnées sont exécutées les unes après les autres. Le sablier indique qu’une étape est en cours de calcul.

Exécution d’un cas

Les étapes sont représentées avec :

  • pour les exécutions terminées avec succès,
  • pour les exécutions en échec,
  • pour les exécutions interrompues (manque de temps par exemple),
  • pour les cas-tests avec des résultats inexacts (NOOK_TEST_RESU).

Les étapes pour lesquelles les résultats précédents sont conservés sont affichées dans une couleur plus foncée, avec un icône ok.

Résultats du calcul

Fenêtre des paramètres d’exécution¶

Les paramètres souvent utilisés peuvent être modifiés dans l’onglet Basique. Le champ Description utilisateur est un champ libre pour identifier un calcul.

L’autre onglet contient les paramètres Avancés. Certains sont dédiés à la soumission des travaux sur serveurs (partition, qos, …) et sont les mêmes que ceux qui peuvent être trouvés dans JobManager.

Les paramètres additionnels sont passées « tels quels » au gestionnaire de travaux. Par exemple : #SBATCH --begin=now+1hour pour Slurm. Voir la documentation de votre gestionnaire de travaux pour les détails.

Suivi du calcul¶

L’utilisateur peut suivre l’exécution à l’aide des outils situés dans la partie basse de l’écran.

Barre d’outils pour le suivi d’un cas

Sortie et messages¶

La partie basse de l’écran permet d’accéder au fichier de sortie de l’exécution et aux messages émis au cours du calcul (informations, alertes, erreurs).

L’onglet Sortie permet de voir les traces produites pour chacun des cas exécutés.

La liste déroulante permet d’afficher un fichier des messages généré pendant l’exécution.

La zone de recherche à droite permet d’effectuer une recherche dans le texte. Le texte cherché est surligné, et l’utilisateur peut consulter les occurrences grâce aux boutons précédent/suivant.

Onglet sortie

L’onglet Messages permet de visualiser les messages d’erreurs, d’avertissement ou d’information générés pendant l’exécution.

L’utilisateur peut filtrer les messages par catégories en activant les boutons correspondants à chacun des types de messages. Il peut également filtrer ces messages par étapes ou par sources en sélectionnant les éléments voulus dans les listes déroulantes.

Onglet messages

Ré-exécuter un cas¶

Il est possible de ré-exécuter un cas en ne ré-exécutant que certaines et en repartant des résultats des précédentes exécutions.

Pour cela, l’utilisateur doit sélectionner l’exécution dont il souhaite réutiliser les résultats, puis sélectionner dans le menu contextuel Utiliser par défaut.

Réutilisation des résultats d’un cas

L’utilisateur peut alors choisir de réutiliser les résultats des premières étapes en choisissant l’option « réutiliser » .

Exécution d’un cas en réutilisant des résultats précédents

Il est ainsi possible de faire plusieurs calculs successifs.

Exécutions successives

Avertissement

Il faut toujours sélectionner le Cas courant (premier noeud de l’arbre en rouge) pour pouvoir lancer un calcul.

Le menu Opérations > Ré-exécuter (dans la Vue cas) permet de ré-exécuter rapidement le Cas courant avec les mêmes paramètres que lors de l’exécution précédente.

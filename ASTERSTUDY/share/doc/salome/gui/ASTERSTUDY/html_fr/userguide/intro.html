Introduction à AsterStudy¶

L’application AsterStudy permet de mettre en données des cas d’étude pour Code_Aster. Elle permet également de lancer les calculs, de suivre l’avancement de ces calculs et d’accéder aux résultats.

Lancement¶

L’application AsterStudy est un module SALOME qui peut être activé en cliquant l’icône de la barre d’outil, ou en sélectionnant le module dans la liste déroulante des modules.

Interface utilisateur¶

Fenêtre principale

L’interface de l’application AsterStudy est composée de deux vues :

  • Menu principal (1),
  • Barres d’outils (2),
  • Et deux vues principales dans la zone centrale (3):
    • La Vue Historique qui permet de voir l’historique d’un cas (3.1),
    • La Vue Cas qui permet de visualiser et d’éditer un cas (3.2).

Vous pouvez sélectionner la vue que vous souhaitez utiliser en sélectionnant l’onglet correspondant à gauche.

Note

Les onglets sont affichés dans la partie gauche de la fenêtre principale par défaut ; leur position peut être modifiée dans la boîte de dialogue Préférences.

Menu principal¶

Les menus sont les suivants :

  • Fichier : opérations générales Nouveau, Ouvrir, Enregistrer, Préférences (opérations standards de SALOME),
  • Éditer : opérations d’édition, suppression, renommage (en fonction de l’élément sélectionné),
  • Opérations : opérations sur le cas, les étapes, les commandes …,
  • Commandes : liste des commandes,
  • Aide : accès à l’aide en ligne.

Barres d’outils¶

De nombreuses opérations sont disponibles via les barres d’outils contextuelles. Les boutons dans les barres d’outils dupliquent les éléments correspondants dans le menu principal et / ou les menus contextuels des autres éléments de l’interface graphique.

Vue Cas¶

Vue Cas

La Vue Cas est composée des éléments suivants :

  • L’onglet Données affiche les détails du cas, liste des étapes et des commandes (1),
  • Le panneau Fichiers affiche les fichiers utilisés en entrée et en sortie par le cas (2),
  • Le panneau Information affiche des informations sur l’élément sélectionné (commande, …) (3),
  • Une zone d’affichage au centre affichant les maillages et les groupes sélectionnés (4),
  • Un panneau permettant d’afficher et d’éditer une commande (5).

Vue Historique¶

Vue Historique

La Vue Historique est composée des éléments suivants :

  • Le panneau Cas affiche la liste des cas (1),
  • Le panneau Fichiers affiche les fichiers utilisés par le cas sélectionné (2),
  • Le panneau Paramètres d’exécution permet d’afficher et éditer les paramètres de calcul du cas sélectionné (3),
  • Le panneau Tableau de bord affiche l’historique d’exécution du cas d’étude (4),
  • Le panneau Messages* affiche les informations sur les exécutions du cas sélectionné (trace, logs, résultats…) (5).

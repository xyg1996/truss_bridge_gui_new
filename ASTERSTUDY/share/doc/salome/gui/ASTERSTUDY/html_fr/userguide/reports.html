Rapports avancés¶

Concepts non utilisés¶

Afin de pouvoir rapidement détecter les concepts qui ne sont pas utilisés dans l’étude courante, il est nécessaire de :

  • Sélectionner le cas courant dans l’onglet Données,
  • Activer le menu Opérations > Concepts non utilisés.

Cette action va ouvrir une boîte de dialogue qui liste tous les concepts qui ne sont pas référencés par d’autre concept. L’utilisateur peut sélectionner tous ou certains de ces concepts et appuyer sur le bouton Supprimer pour nettoyer l’étude.

Concepts non utilisés

Avertissement

Le dernier concept de l’étude est toujours, par définition, non utilisé.

Rapport de validité¶

L’utilisateur peut contrôler la validité d’une étape en cours de création, directement dans l’onglet Données : chaque commande invalide est affichée en rouge. Une étape est considérée non valide si un de ces concepts est invalide.

L’objet peut être invalide pour les raisons suivantes :

  • L’objet est syntaxiquement invalide (un mot-clé nécessaire de la commande n’est pas renseigné),
  • Dépendance cassée (cela peut arriver si, par exemple, un concept référencé par un autre a été supprimé),
  • Conflit de nom (un même nom est utilisé plusieurs fois).

L’utilisateur peut voir la validité de l’étape sélectionnée, ou celle du cas, en utilisant le menu Opération > Rapport de validité.

Ce menu va ouvrir une boîte de dialogue qui affichera, dans une table, le statut de chacun des concepts. L’utilisateur peut également voir si un concept est utilisé (référencé par d’autres concepts) ou pas.

Groupes impliqués¶

Les groupes de mailles utilisés par certains concepts peuvent être vue de la façon suivante :

  • Sélectionner un concept dans l’onglet Données,
  • Activer le menu contextuel Groupes impliqués, ou
  • Presser le bouton dans le panneau d”Information.

Cela ouvrira un panneau Résumé dans la partie basse de la fenêtre de l’application. Ce panneau liste, dans une table, tous les groupes de mailles utilisés par le concept. Chaque cellule correspond à une utilisation dans la commande avec un code de couleur unique. Une * utilisation* est le contexte du concept dans lequel des groupes de mailles particuliers (un groupe séparé ou une liste de groupes) sont utilisés.

Dans l’exemple ci-dessous, le concept bc produit par la commande AFFE_CHAR_MECA déclare deux utilisations :

  • DDL_IMPO / 0, qui utilise les groupes Cont_mast, Cont_slav, et
  • DDL_IMPO / 1, qui utilise les groupes Sym_x, Sym_y.
bc = AFFE_CHAR_MECA(
        DDL_IMPO=(_F(DX=1.0,DY=1.0,DZ=1.0,
                     GROUP_MA=('Cont_mast', 'Cont_slav')),
                  _F(DX=2.0,DY=2.0,DZ=2.0,
                     GROUP_MA=('Sym_x', 'Sym_y'))),
        MODELE=model)

Pour des raisons de performances, les utilisations sont affichées dans le panneau Résumé par page. La taille d’une page est définie dans Préférences de l’application. L’utilisateur peut naviguer entre les pages en utilisant les boutons Précédent et Suivant.

L’utilisateur peut afficher dans le maillage les utilisations sélectionnées en :

  • Pressant l’icône affiché sur la légende de l”utilisation dans le panneau,
  • Choisissant le menu contextuel Voir les groupes,
  • Pressant l’icône de la barre d’outils.

L’utilisateur peut également supprimer les utilisations sélectionnées de la vue de maillage en :

  • Pressant l’icône affiché sur la légende de l”utilisation dans le panneau,
  • Choisissant le menu contextuel Cacher les groupes,
  • Pressant l’icône de la barre d’outils.

Lorsqu’une utilisation est affichée, les groupes de mailles impliqués sont affichés avec la couleur de l”utilisation dans la vue de maillage. Cela permet ainsi de distinguer les différentes utilisations dans la vue.

Avertissement

Le même groupe de mailles peut être affecté à différentes utilisations. Dans la vue du maillage, un groupe sera coloré avec une seule couleur, très probablement celle affectée à l’utilisation la plus faible du panneau. L’utilisateur sera averti de cette situation par un message.

Panneau des groupes impliqués

Résumé d’analyse¶

Pour les commandes appartenant à la catégorie Analysis, le menu Groupes impliqués n’est pas disponible. Au lieu de cela, l’utilisateur peut ouvrir le panneau Résumé en utilisant le menu contextuel Résumé d’Analyse ou en appuyant sur le bouton dans le panneau Information.

Pour ces commandes, le panneau Résumé affiche tous les groupes de mailles impliqués dans les commandes dont la commande sélectionnée de la catégorie Analyse dépend, directement ou indirectement.

Panneau résumé d’analyse

Dans ce mode, le panneau Résumé groupe toutes les utilisations par catégories. L’utilisateur peut filtrer les utilisations d’une certaine catégorie en la sélectionnant dans la liste déroulante du panneau.

Onglet Fichiers¶

Description¶

Le panneau Fichiers de la vue Cas permet de gérer les fichiers utilisés en entrée et en sortie par le cas.

Panneau Fichiers

L’utilisateur peut voir et éditer les fichiers en sélectionnant un cas, une étape ou une commande dans le panneau Données. Dans tous les cas, le panneau Fichiers affiche tous les fichiers référencés par l’élément sélectionné.

La colonne Unité affiche l’identifiant unique du fichier dans l’étude.

La colonne Mode affiche le type du fichier : in, out, ou inout.

La colonne Présent indique si le fichier existe ou non sur le disque.

La colonne Embarqué indique si le fichier est intégré dans le fichier de sauvegarde du cas.

En mode graphique les fichiers sont ajoutés automatiquement par l’application. Pour chaque fichier le panneau Fichiers affiche les commandes référençant ce fichier (voir image ci-dessus).

En mode Texte, l’utilisateur doit ajouter manuellement les fichiers qu’il a déclaré lorsqu’il a décrit les commandes. Pour cela il peut utiliser les boutons Ajouter fichier et Supprimer fichier de la barre d’outils du panneau Fichiers.

Dans l’environnement Gnome (les modules Python Gio et GLib sont nécessaires pour cette fonctionnalité), l’explorateur de fichiers permet de sélectionner des fichiers sur un serveur distant. Par défaut, les serveurs distants ne sont pas automatiquement connectés au démarrage. L’utilisateur peut changer ce comportement dans Fichier > Préférences. Pour connecter manuellement les serveurs distants, utilisez le menu Opérations > Connecter les serveurs distants. Les systèmes de fichiers distants sont automatiquement démontés quand l’Étude est fermée. Les serveurs distants sont déclarés dans les ressources du module SALOME JobManager.

Éditer un fichier¶

Il est possible d’éditer en sélectionnant un fichier, puis en cliquant sur le bouton Editer.

Éditer un fichier

Fichiers embarqués¶

Les fichiers d’entrées de l’étude peuvent être embarqués dans le fichier de l’étude (format HDF). Cela a pour avantage de pouvoir déplacer le fichier d’une machine à une autre sans se soucier de devoir déplacer d’autres fichiers.

Embarquer un fichier¶

Pour embarquer un fichier, l’utilisateur sélectionne le fichier dans le panneau Fichiers, puis coche la case embarquer dans le menu contextuel.

Embarquer un fichier à partir du menu contextuel

Cette action peut également être réalisée sur le panneau d’édition du fichier.

Embarquer un fichier à partir du panneau d’édition

Détacher un fichier¶

Pour détacher un fichier, il faut procéder de la même manière pour décocher la case. L’application demandera alors à l’utilisateur où il souhaite mettre le fichier détaché.

Ouvrir avec¶

L’utilisateur peut ouvrir un fichier de données pour le visualiser ou l’éditer dans AsterStudy ou dans une application externe en effectuant les opérations suivantes :

  • Ouvrir dans l’éditeur : ouvre le fichier sélectionné en mode édition dans l’éditeur interne ou dans un éditeur externe, selon les préférences choisies,
  • Ouvrir dans ParaVis : ouvre le fichier MED sélectionné dans le module ParaVis de SALOME,
  • Ouvrir avec : permet à l’utilisateur de choisir le programme à utiliser pour ouvrir le fichier sélectionné.

Résumé¶

Le résumé des fichiers utilisés dans une étude peut être affiché dans le panneau Résumé des fichiers dans la vue Historique. L’utilisateur peut voir tous les fichiers, et si nécessaire, modifier rapidement un chemin.

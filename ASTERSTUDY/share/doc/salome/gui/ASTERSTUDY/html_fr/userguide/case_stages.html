Cas et étapes¶

Selon les termes de Code_Aster, un cas d’étude est décrit comme une séquence de commandes. Cette séquence est transmise au code de calcul via un ou plusieurs fichiers de commandes (normalement avec des extensions de la forme « .com », par exemple aaa.comm, aaa.com1, aaa.com2…).

Dans l’application AsterStudy, un cas d’étude est représenté comme une liste d’étapes, pour chaque étape un fichier de commande séparé sera créé.

Cas¶

Lorsqu’une étude est créée, un cas vide est automatiquement ajouté. Le cas représente une étude Aster dans l’application.

Le contenu du cas est complété selon les besoins par diverses opérations. En plus du Cas courant qui représente le cas en cours d’édition par l’utilisateur, d’autres types de cas existent :

  • Cas exécutés : les cas contenant des résultats d’un calcul Code_Aster,
  • Cas sauvegardés : copies de sauvegarde du Cas courant.

Pour plus de détails sur les fonctionnalités des cas, voir le chapitre Gestion des cas.

Étapes¶

Un cas est composé d”étapes représentant les différentes phases du calcul Aster.

Mode graphique et mode texte des étapes¶

Les étapes peuvent être gérées soit en mode graphique soit en mode texte.

Le mode graphique permet de gérer les étapes en renseignant les étapes via des panneaux de saisie dans l’interface de l’application.

Le mode texte laisse l’utilisateur saisir les commandes Aster. Ce mode peut, par exemple, être utilisé pour utiliser des fonctionnalités de Code_Aster.

Les étapes en mode graphique sont représentées dans l’onglet Données par l’icône , et celles en mode texte par l’icône .

Important

Par défaut les étapes sont créées en mode graphique.

Il est possible de changer le mode d’une étape via le menu contextuel ou le menu Opérations.

Avertissement

Un message d’avertissement sera affiché si l’application ne peut pas convertir une étape du mode texte vers le mode graphique.

Ajouter une étape¶

Pour ajouter une étape, placer vous dans la vue cas au choix :

  • cliquer sur le bouton Ajouter une étape de la barre d’outil,
  • utiliser le menu Opérations > Ajouter une étape,
  • le menu contextuel Ajouter une étape sur le cas dans l’onglet Données,
  • utiliser le raccourci clavier Ctrl+Maj+N.

Les nouvelles étapes sont nommées automatiquement par l’application. Il est possible de renommer une étape en la sélectionnant dans l’onglet Données et en utilisant le menu contextuel Renommer, ou en pressant le raccourcis F2.

Importer une étape¶

Il est possible d’importer un fichier de commandes Aster en tant qu’étape du cas.

Pour cela, vous pouvez soit :

  • cliquer l’icône Ajoute une étape à partir d’un fichier de la barre d’outil,
  • utiliser le menu Opérations > Ajoute une étape à partir d’un fichier,
  • utiliser le menu contextuel Ajouter une étape à partir d’un fichier sur le cas dans l’onglet Données,
  • utiliser le raccourci clavier Ctrl+Maj+I.

Important

L’application analyse le fichier de commandes et crée une étape en mode graphique à partir du contenu du fichier. Si le fichier contient des commandes ne pouvant être interprétées par l’application l’étape sera créée en mode texte avec le contenu du fichier.

Il est également possible d’importer un fichier de commandes Aster en tant qu’étape en mode texte, c’est-à-dire sans la convertir en mode graphique. Pour cela :

  • utiliser le menu Opérations > Ajoute une étape texte à partir d’un fichier,
  • utiliser le menu contextuel Ajouter une étape à partir d’un fichier sur le cas dans l’onglet Données.

Importer une étape depuis un modèle¶

Il est possible de créer une étape en important un modèle prédéfini de commandes Aster. Les modèles peuvent être créées par les utilisateurs dans le répertoire ~/.config/salome/asterstudy_templates. Ces fichiers apparaitront dans :

  • les menus Opérations > Ajouter une étape depuis un modèle et Opération > Ajouter une étape texte depuis un modèle,
  • les sous-menus Ajouter étape depuis un modèle et Ajouter étape texte depuis un modèle du menu contextuel sur le cas dans l’onglet Données.

Important

Des templates prédéfinis peuvent-être fournis avec l’application AsterStudy.

Ajouter une étape avec un assistant¶

Une autre façon d’ajouter une étape au cas d’étude consiste à utiliser un assistant de calcul.

Un assistant de calcul est une étude paramétrée prédéfinie, disposant d’une interface utilisateur, permettant à l’utilisateur de saisir les paramètres et de pouvoir rapidement créé un cas de calcul.

Plusieurs assistants de calcul sont fournis avec l’application AsterStudy. Les assistants personnalisés de l’utilisateur sont localisés dans le répertoire ~/.config/salome/asterstudy_assistants. Ces assistants apparaissent dans :

  • le menu Opérations > Ajouter une étape avec un assistant,
  • les sous-menus Ajouter étape avec un assistant du menu contextuel sur le cas dans l’onglet Données.

Pour en savoir plus sur les assistants de calcul (en particulier, comment créer vos propres assistants), consulter Guide de l’assistant de calcul

Exporter une étape¶

De la même façon, il est possible d’exporter une étape vers un fichier de commandes Aster :

  • cliquer l’icône Exporter le fichier de commandes de la barre d’outils,
  • utiliser le menu Opération > Exporter le fichier de commandes,
  • utiliser le menu contextuel Exporter le fichier de commandes sur l’étape à exporter dans l’onglet Données,
  • utiliser le raccourci clavier Ctrl+Maj+E.

Si l’étape est en mode graphique, le fichier de commande est générée à partir des commandes définies dans l’étape, dans le cas d’une étape en mode texte, le contenu de l’étape est directement exporté vers le fichier.

Renommer l’étape¶

Les étapes sont ajoutées à l’étude avec un nom par défaut. Pour renommer une étape, sélectionner la dans l’onglet Données et :

  • Utiliser le menu principal Edition > Renommer,
  • Utiliser le menu contextuel Renommer,
  • Presser la touche F2.

Supprimer une étape¶

Une étape peut être supprimée du cas en utilisant le bouton de la barre d’outils , le menu principal, ou le raccourci (Del).

Avertissement

La suppression d’une étape, supprimera automatiquement toutes les étapes suivantes.
